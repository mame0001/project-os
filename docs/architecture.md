# 架構說明 / Architecture

Project-OS 是一套輕量的個人/小團隊專案管理架構，把三個現成工具各司其職、互相連結：

```
        ┌──────────────────────────────────────────────┐
        │                  一個專案                       │
        │            ONE PROJECT, ONE NAME               │
        └──────────────────────────────────────────────┘
              │                │                 │
       ┌──────▼─────┐   ┌──────▼─────┐    ┌──────▼─────┐
       │   Notion   │   │  Obsidian  │    │   Drive    │
       │  執行 Exec  │   │  知識 Know  │    │ 成果 Deliv  │
       │  進度/任務  │   │  筆記/計畫  │    │  最終檔案   │
       └──────┬─────┘   └──────┬─────┘    └──────┬─────┘
              │  obsidian://   │  url 欄位         │
              └────────────────┴──────────────────┘
                       三方互相留連結
```

## 設計原則 / principles
1. **單一職責**：執行→Notion、知識→Obsidian、成果→Drive，不混放。
2. **同名綁定**：三邊用同一個 `專案-客戶-主題` 名字。
3. **雙向連結**：任一邊都能一鍵跳到另外兩邊。
4. **成果不散落**：所有交付物進 Drive `02_成果輸出/`。
5. **生命週期分流**：進行中放 `00_進行中/`，結案移到根目錄。

## 為什麼是這三個？ / why these three
- Notion 的關聯式資料庫 + 視圖適合「動態進度與任務」。
- Obsidian 的純 Markdown + 雙向連結適合「長存知識」，不被平台綁架。
- Drive 適合「大型/二進位檔案」與分享。

## 安裝後的資料流 / data flow
1. `new-project.sh` 建 Obsidian 儀表板（含 obsidian:// 自連結）。
2. Notion 建同名卡，貼上 Obsidian / Drive 連結。
3. 做事：任務在 Notion，知識在 Obsidian，檔案進 Drive。
4. 結案：Notion=Done、Drive 移根、Obsidian 沉澱。

完整 SOP 見安裝後 vault 內的 `90_系統_System/專案生命週期說明書.md`。
